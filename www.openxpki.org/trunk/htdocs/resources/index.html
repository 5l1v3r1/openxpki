Resources

OpenXPKI uses the sourecForge.net platform for various things. The major aspects why we choose SourceForge.net is the stability and professional level which the SourceForge.net staff provides to the projects.

  • OpenXPKI at SourceForge
  • Mailing Lists
  • Bug Tracking
  • RFEs - Feature requests
  • Subversion (Version Control System)
  • ViewCVS (but with Subversion content) updates with each commit
  • WebSVN at www3 updates once in 5 min

Subversion structure

We use the version control system from SourceForge. It contains the most up-to-date development code. The actual structure under / (svn root) is the following one:

/trunk/
perl-modules/core/trunk/
Core for server part
clients/perl/
OpenXPKI-Client/
Base class for actual clients
OpenXPKI-Client-CLI/
Command line interface to a locally running OpenXPKI daemon
OpenXPKI-Client-HTML-Mason/
Web interface to a locally running OpenXPKI daemon
OpenXPKI-Client-SCEP/
SCEP tools
OpenXPKI-Client-SOAP-Lite/
SOAP-Lite tools
deployment/
Deployment tools
i18n/
Internationalization of the user interface
package/
debian/
Tools to build fresh Debian Linux package
freebsd/
FreeBSD port and tools to update it
/www.openxpki.org/trunk/
src/
Sources for the web pages
htdocs/
The web pages themselves
update_www_openxpki_org
A script to update the web servers
